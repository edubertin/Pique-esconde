# Internacionalização

## Objetivo

Preparar o app para múltiplos idiomas sem adicionar complexidade pesada no MVP.

O idioma principal do produto continua sendo `pt-BR`, mas a estrutura já deixa espaço para `en-US` e futuros idiomas.

## Estrutura

```txt
apps/mobile/src/i18n/
  index.ts
  pt-BR.ts
  en-US.ts
```

## Uso no Código

Use o helper `t()` para textos de interface:

```ts
import { t } from '@/src/i18n';

t('lobby.invite');
t('result.summaryHiders', { name: player.nickname });
```

## Regras

- Novos textos de UI devem entrar primeiro em `pt-BR.ts`.
- `en-US.ts` deve espelhar todas as chaves de `pt-BR.ts`.
- Não deixar textos novos soltos em telas ou componentes quando eles forem texto de interface.
- Pode manter dados internos, ids e valores mock fora do i18n.
- O idioma ativo fica fixo em `pt-BR` no MVP.
- A troca dinâmica de idioma fica para etapa futura, quando houver tela de configuração ou detecção de idioma do dispositivo.

## Escopo Inicial

Primeira migração:

- Home.
- Criar sala.
- Entrar na sala.
- Permissão de localização.
- Lobby.
- Fase de esconder.
- Radar.
- Captura.
- Resultado.
- Card social.
- Regras.
- Componentes compartilhados de avatar, lista de jogadores e voltar.

## Decisão

Manter solução leve e tipada no MVP, sem `i18next` ou biblioteca externa.

Se o app avançar para distribuição internacional, avaliar:

- detecção automática de idioma do dispositivo;
- persistência de idioma escolhido;
- pluralização mais robusta;
- integração com ferramenta de tradução.
